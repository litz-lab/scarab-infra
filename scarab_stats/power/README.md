# 🧩 McPAT–Scarab Format Converter

> **Update**: this directory now also ships an end-to-end pipeline
> (`run_one_simpoint.py` + `aggregate_workload_power.py`) integrated as
> the `./sci --power DESCRIPTOR` subcommand. The pipeline reads scarab's
> native `power.stat.0.csv` directly (no manual `power.pkl`) and
> produces simpoint-weighted whole-workload power & energy summaries.
>
> ## Quick usage
> ```
> ./sci --power agent_sim
> ```
> Output: `<root_dir>/simulations/<experiment>/power_summary.csv` with
> columns `suite, subsuite, workload, config, n_simpoints_with_power,
> weight_covered, avg_runtime_dynamic_W, avg_leakage_W, avg_total_W,
> weighted_cycles, sim_time_s, energy_J`.
>
> ## Pipeline files
> - `parse_power_stat.py` — parses scarab's `power.stat.0.csv` totals
>   into the dict format `converter.py` expects (replaces `power.pkl`)
> - `aggregate_workload_power.py` — walks all simpoints under a
>   simulation experiment, runs the full PR 178 conversion + `mcpat`
>   binary per simpoint (cached under `<sim_dir>/power/`), and produces
>   a weighted whole-workload summary CSV using simpoint weights from
>   `workloads/workloads_db.json`. This is what `./sci --power`
>   invokes.
>
> The original PR 178 manual workflow (described below) is still
> available as the underlying conversion library.

---

This utility provides a lightweight Python interface to bridge Scarab simulation outputs and McPAT power modeling inputs.
It automates the conversion pipeline between Scarab’s parameter/statistics files and McPAT’s XML format.

---

## 📁 Structure

```
.
├── converter.py          # Core conversion logic (Converter class)
├── client.py             # Command-line interface for using Converter
├── xml/
│   ├── template.xml      # McPAT XML template
│   └── mcpat_infile.xml  # Generated final XML (Converter output)
├── json/                 # Generated JSON intermediate files
│   ├── params_table.json
│   ├── stats_table.json
│   └── mcpat_structure.json
├── scarab_output/        # Scarab simulation output (Converter input)
│   ├── PARAMS.out        # Scarab simulation architecture parameters
│   └── power.pkl         # Scarab simulation runtime power statistics
└── mcpat
```

## 🚀 Usage

The main entry point of the workflow is client.py, which provides subcommands for each stage of the McPAT–Scarab conversion process. You can either execute the entire pipeline in a single command or run each step individually for finer control. To generate the final McPAT XML input in one go, simply run:
```
python client.py all
```

### 1️⃣ Generate initial JSON files from McPAT XML template
```
python client.py generate-json
```
 - input: ./xml/template.xml
 - output: ./json/

The input to this step is a sample McPAT XML template that defines the hierarchical organization and parameters of the modeled architecture. When the generate-json command is executed, the tool parses this XML and generates three JSON files: mcpat_structure.json, which captures the hierarchical tree structure of the XML; params_table.json, which stores the static architectural parameters describing the configuration properties; and stats_table.json, which represents the dynamic power-related data collected from simulation.

### 2️⃣ Update JSON with Scarab PARAMS file
```
python client.py update-params
```
 - input: ./scarab_output/PARAMS.out
 - output: ./json/params_table.json

This step uses the PARAMS.out file generated by Scarab to update and adjust the corresponding values in params_table.json.

### 3️⃣ Update JSON with Scarab STATS file
```
python client.py update-stats
```
 - input: ./scarab_output/power.pkl
 - output: ./json/stats_table.json

This step updates the dynamic power statistics in ./json/stats_table.json using a dictionary of runtime data. This dictionary should be manually generated after running the Scarab simulation and should contain the relevant power-related metrics, which are then written into the corresponding fields of the JSON file.

### 4️⃣ Generate final McPAT XML file
```
python client.py generate-xml
```
 - input: ./json
 - output: ./xml/mcpat_infile.xml

This step combines the data from the parameter table and the statistics table according to the predefined hierarchical structure, and regenerates the complete McPAT XML input file. The resulting XML serves as the finalized input for McPAT’s power, area, and timing analysis.

---

## 🚀 McPAT Launch

After the final step, you can run McPAT manually:

```
./mcpat -dump_design -infile xml/mcpat_infile.xml -print_level 5 > scarab.out
```
